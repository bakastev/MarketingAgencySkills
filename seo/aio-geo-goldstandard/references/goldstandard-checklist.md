# Gold Standard Checklist (AIO / GEO / AEO)

Strict **pass / conditional / fail** gates. Pair with `SKILL.md` **Non-negotiables** and **Quick gates**.

## Content

1. Primary intent is explicit in H1 or the opening block.
2. Opening uses a **declarative** claim in the first 1–2 sentences (no throat-clearing).
3. The lead (~120 words) avoids **hedging-as-the-main-move** and **price-dominated** openings unless intent is pricing/finance/commerce.
4. Each major section begins with a **direct answer** (target: first 40–60 words).
5. Heading strategy is **deliberately flat** OR **fully purposeful** (each H2 = one clear question). **Not** a thin set of three to four arbitrary H2s.
6. At least one concrete **information gain** element exists (data, method, first-hand insight, synthesis).
7. Filler is minimized; claims are specific and testable where possible.
8. Strong scannability (lists, tables, short paragraphs).
9. Obvious follow-up questions are reflected in H2/H3 structure.

## Entity and trust

10. Primary and supporting entities are explicit and consistent.
11. Author or organization context is clear when the topic warrants it.
12. High-impact claims include evidence context (source, method, or lived experience).
13. The page reads as a credible **entity home** for its primary topic.

## Technical and code

14. Critical answer content is **server-rendered** or otherwise reliably crawlable.
15. Structured data is present, appropriate to intent, and syntactically valid.
16. JSON-LD values **match** visible on-page text.
17. Internal links support the topic cluster and canonical hubs.
18. Update / freshness metadata is present and honest.

## AI retrieval

19. Chunks are extraction-friendly (clear boundaries, atomic facts).
20. Key claims can be quoted verbatim without ambiguity.
21. FAQ-style blocks appear only when visible Q&A exists on the page.
22. `llms.txt` is optional auxiliary signal—not the primary strategy.
23. No deceptive optimization (fake dates, synthetic authority, fabricated references).

## Scoring

- **PASS**: ≥21 checks pass **and** zero critical failures (14, 15, 16, 23) **and** no **Quick gate** failures from `SKILL.md`.
- **CONDITIONAL**: 18–20 checks pass, zero critical failures, and any remaining issues are explicitly listed with fixes.
- **FAIL**: <18 pass; **or** any critical failure; **or** any **Quick gate** failure unresolved.
