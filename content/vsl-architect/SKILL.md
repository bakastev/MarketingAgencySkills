# VSL Architect

Erstellt strategisch fundierte, avatar-getriebene Video Sales Letters — vom Script über die Hook-Bibliothek bis zum Produktions-Blueprint. Jedes Script-Element ist auf Avatar-Sections zurückführbar, jedes Framework ist preisklassen- und segment-gerecht.

**VSL 2026 ist kein isoliertes Video — es ist ein Systembestandteil der Conversion-Pipeline.**

---

## Core Principles

1. **System über Kampagne** — Eine VSL ist kein One-Off. Sie ist Teil eines kontinuierlich lernenden Systems: Build → Measure → Learn → Iterate. Isolierte Taktiken tot.
2. **Avatar-Evidenz (STRIKT)** — Jedes Script-Element MUSS auf eine Avatar-Section zurückführbar sein. Keine Behauptungen ohne Daten-Bezug.
3. **Preisklasse bestimmt alles** — Länge, Framework, Produktionstiefe und CTA-Architektur ergeben sich aus dem Preispunkt. Eine 30-Min-VSL für einen 7€-Lead Magnet ist Budget-Verschwendung.
4. **Mobile-First + Silent Video** — 82,9% der Views kommen von Mobile. 18,7% mobile Conversion vs. 28,9% Desktop. Silent Video Optimization ist Pflicht, nicht Kür.
5. **Human-in-the-Loop bei KI** — KI für repetitive Tasks (Varianten, Captions, Personalisierung). Mensch für kreative Oberhoheit (Story, Empathie, Brand Voice). Kein Uncanny Valley.
6. **Pipeline-Qualität statt Vanity Metrics** — Views bedeuten nichts. View-Through-Rate, Engagement-Heatmaps und Conversion-Rate sind die Metriken die zählen.
7. **Rechtssicher 2026** — AI Act (ab 02.08.2026), BFSG/WCAG 2.1 AA, Kennzeichnungspflicht. Non-Compliance kostet bis zu 35M€. Jede VSL muss compliance-ready sein.
8. **Segment-aware** — Verschiedene Segmente brauchen verschiedene VSLs. Ein 55-jähriger Unternehmer braucht eine andere Hook als ein 28-jähriger Junior-Marketer.
9. **Authentizität schlägt Polished** — Hochglanz-Studioproduktion wirkt 2026 distanziert. Smartphone-Ästhetik mit starkem Script > teure Produktion mit schwachem Script.
10. **Composio-Integration** — Finaler Script-Upload und -Distribution über Google Drive/YouTube via Composio.

## Module 1: VSL Strategy & Framework Selection

### 1.1 Preispunkt → Länge → Framework Matrix

| Preisklasse | Länge | Framework-Priorität | Hook-Strategie | CTA-Typ |
|-------------|-------|--------------------| -------------|---------|
| **Free Lead Magnet** | 1–3 Min | PAS (kurz), Story-Teaser | Curiosity Gap, Specific Result | Soft CTA (Download/Register) |
| **Low Ticket** (1–200€) | 1–5 Min | PAS + Social Proof | Negative Bias, Specific Result | Direct CTA + Urgency |
| **Mid Ticket** (200–1.000€) | 5–12 Min | PAS + Story + Objection Handling | All 4 Hook-Typen | Stack CTA + Guarantee |
| **High Ticket** (1.000€+) | 10–30 Min | Full Story + Authority + Objection | Curiosity Gap + Negative Bias | Consultation CTA + Risk Reversal |

### 1.2 Framework-Auswahl

**PAS (Problem-Agitation-Solution) — Grundgerüst für ALLE VSLs:**
- Problem: Sprich den Schmerz des Avatars direkt an
- Agitation: Mach den Schmerz greifbar (Konsequenzen, Kosten des Nichthandelns)
- Solution: Präsentiere dein Offer als logische Antwort

**Dr. Gundry-Ansatz (Bildung + Entdeckung):**
- Start mit einer überraschenden Behauptung/Fakten
- Mythos entlarven ("Das was dir erzählt wurde, ist falsch")
- Eigene Entdeckungsgeschichte
- Wissenschaftliche/Expert-Backing
- Länger, bildungsorientiert → Mid/High Ticket

**Sabri Suby-Ansatz (Story + Schock + Pattern Interrupt):**
- Story-Opener mit persönlichem Tiefpunkt
- Schock-Moment ("Ich habe X€ verschwendet")
- Pattern Interrupt (visueller Bruch, Sprecherwechsel)
- Lead Magnet als Low-Barrier Einstieg
- Ideal für Lead-Gen und Low/Mid Ticket

**V-Shred-Ansatz (Social Proof + Micro-Commitment):**
- Heavy Social Proof direkt am Anfang
- Quiz/Assessment als Micro-Commitment
- Personalisierte VSL basierend auf Quiz-Ergebnis
- Aggressive Social Proof Stack
- Ideal für D2C und Mid Ticket

### 1.3 Segment → Hook-Strategie Mapping

```yaml
segment_hook_strategy:
  segment_analysis:
    - segment: "[Name aus Avatar]"
      primary_pain: "[Top Pain]"
      dominant_belief: "[Hindering Belief]"
      hook_category_primary: "Negative Bias | Curiosity Gap | Visual Disruption | Specific Result"
      framework_fit: "PAS | Gundry | Suby | VShred"
      vsl_length: "[aus Preisklasse]"
      tone: "[professionell|locker|provokant|empathisch]"
```

**Regel:** Pro Segment mindestens eine eigene VSL-Variante. Shared Elements nur bei Avatar-Überschneidung ≥60%.

## Module 2: Script Architecture

### 2.1 Struktur-Master-Template (an Länge anpassbar)

```
┌─────────────────────────────────────────────┐
│  HOOK (0–3s)                                │
│  Open Loop → Scroll-Stopper                │
├─────────────────────────────────────────────┤
│  RAPPORT (3–15s)                            │
│  "Ich bin wie du" — Identifikation          │
├─────────────────────────────────────────────┤
│  PROBLEM (15–45s)                           │
│  Schmerz benennen — Avatar Pain-Referenz    │
├─────────────────────────────────────────────┤
│  AGITATION (45s–2min)                       │
│  Konsequenzen, Kosten, Emotion             │
├─────────────────────────────────────────────┤
│  SOLUTION INTRO (2–3min)                    │
│  "Es gibt einen Weg" — Pattern Interrupt    │
├─────────────────────────────────────────────┤
│  HOW IT WORKS (3–6min)                      │
│  3-Schritt-Prozess / Framework             │
├─────────────────────────────────────────────┤
│  SOCIAL PROOF (6–8min)                      │
│  Testimonials, Case Studies, Zahlen         │
├─────────────────────────────────────────────┤

│  Top 3 Einwände direkt adressieren          │
├─────────────────────────────────────────────┤
│  CTA STACK (10–12min)                       │
│  Value Stack + Guarantee + Urgency          │
├─────────────────────────────────────────────┤
│  CLOSE (12–13min)                           │
│  Finaler CTA + Last Chance                 │
└─────────────────────────────────────────────┘
```

** timings sind Richtwerte für eine 12-Min Mid-Ticket VSL. Skaliere proportional.**

### 2.2 Hook (0–3s) — Open Loop

**Ziel:** Zuschauer in den ersten 3 Sekunden captivieren. Open Loop erstellen der erst am Ende geschlossen wird.

**4 Hook-Kategorien mit Mechanik:**

| Kategorie | Mechanik | Avatar-Bezug | Beispiel-Pattern |
|-----------|----------|-------------|------------------|
| **Negative Bias** | Verlustangst aktivieren | Anxiety (E), Belief System (G) | "Du verlierst jeden Tag X€, weil..." |
| **Curiosity Gap** | Informationsdefizit schaffen | Pain (E), JTBD (C) | "Der eine Fehler, der 90% machen..." |
| **Visual Disruption** | Musterunterbrechung | Belief System (G) — hindering | [Schockbild/kontrastiver Schnitt] |
| **Specific Result** | Greifbarkeit schaffen | Desired Outcome (E), Gain (E) | "Wie [Kunde] in [Zeit] [Ergebnis] erreicht hat" |

**Script-Regeln für Hooks:**
- Max. 8 Worte in den ersten 3 Sekunden
- Kein Firmenlogo, keine Einleitung ("Hallo, ich bin...")
- Direkt beim Schmerz oder Ergebnis einsteigen
- Open Loop MUSS im Script aufgelöst werden

### 2.3 Problem/Agitation (Passthrough)

**Problem (Avatar-Referenz: Pain Section E):**
- Schmerz exakt in den Worten des Voice of Customer (Section H) benennen
- Konkret, nicht abstrakt: "Du verbringst 15 Stunden pro Woche mit..." statt "Du hast viel zu tun"
- Avatar-Evidenz-Tag: `[Avatar: Pain #X]` im Script

**Agitation (Avatar-Referenz: Anxiety Section E + Belief System G):**
- Emotionale Eskalation: Was passiert wenn nichts passiert?
- Finanzielle Kosten quantifizieren: "Das kostet dich X€ pro Jahr"
- Psychische Kosten: Stress, Schlaflosigkeit, Beziehungsprobleme
- Zukunftsszenario malen: "In 6 Monaten wird das noch schlimmer, weil..."
- Avatar-Evidenz-Tag: `[Avatar: Anxiety #X + Belief "Zitat"]`

### 2.4 Solution/Benefit

- Lösung als logische Konsequenz des Problems präsentieren
- 3-Schritt-Prozess (Menschen lieben 3er-Gruppen):
  1. Schritt: Was sofort passiert
  2. Schritt: Was sich in Woche X verändert
  3. Schritt: Das Endresultat
- Benefits, nicht Features: "Du sparst 10 Stunden pro Woche" statt "Wir nutzen KI-Automatisierung"
- Avatar-Evidenz-Tag: `[Avatar: Desired Outcome #X]`

### 2.5 Social Proof

**Proof-Stack (mindestens 3 Ebenen):**
1. **Quantitativ:** "X Kunden", "Y% Erfolgsquote", "Z€ eingespart"
2. **Qualitativ:** 1–2 Testimonials die den Avatar-Pain widerspiegeln
3. **Autoritativ:** Expert-Zitate, Zertifikate, Partner-Logos, Presse

**Regeln:**
- Testimonials MÜSSEN den gleichen Avatar-Schmerz adressieren
- Zahlen müssen überprüfbar sein (keine Fake-Stats)
- Bei Mid/High Ticket: Case Study mit Before/After/Process
- Avatar-Evidenz-Tag: `[Avatar: Segment "Name" + Pain #X]`

### 2.6 Objection Handling

**Top-3-Objection-Methodik:**
1. **Pre-frame:** "Du denkst jetzt bestimmt..." (Einwand vorwegnehmen)
2. **Empathize:** "Das ist verständlich, ich dachte auch so."
3. **Reframe:** "Aber dann habe ich entdeckt..."

**Häufige Einwände und Patterns:**
```yaml
objections:
  zu_teuer: "Preis relativieren: Kosten des Nichthandelns vs. Investition"
  keine_zeit: "Zeit-Argument: Inkludierte Zeitersparnis als ROI"
  funktioniert_nicht: "Proof-Stack: Testimonials + Garantie"
  brauche_ich_nicht: "Problem-Agitation wiederholen: Konsequenzen"
  zu_komplex: "Einfachheit betonen: 3-Schritt-Prozess + Support"
```

### 2.7 CTA Stack

**Elemente (für Mid/High Ticket):**
1. **Value Recap:** Kurze Zusammenfassung was der Nutzer bekommt
2. **Offer:** Klarer Preis + Was inkludiert ist
3. **Bonus Stack:** Zusätzliche Boni (verschaffen wahrgenommenen Mehrwert)
4. **Guarantee:** Geld-zurück-Garantie (Mindset: Risiko-Umkehr)
5. **Urgency/Scarcity:** Grund WARUM jetzt (echter Grund, kein Fake-Countdown)
6. **Clear Instruction:** "Klicke auf den Button unter diesem Video"

**CTA-Regeln:**
- EIN CTA pro VSL, nicht drei verschiedene
- Button-Text action-orientiert: "Jetzt kostenlos starten" nicht "Mehr erfahren"
- Bei Free Lead Magnet: Soft CTA, kein Preis erwähnen
- Bei High Ticket: Konsultations-CTA, kein direkter Kauf

### 2.8 Close

- Letzter emotionaler Peak (Story-Moment oder starker Quote)
- Open Loop aus dem Hook schließen
- Finaler CTA mit leichter Variation (nicht wörtliche Wiederholung)
- "P.S." Pattern: Nach dem inoffiziellen Ende noch einen Bonus-Hinweis

## Module 3: Hook Library

### 3.1 Hook-Generierungs-Regeln

**Pro Segment:**
- Mindestens 20 Hooks generieren
- Gleichmäßig über 4 Kategorien verteilt (je ≥4 pro Kategorie)
- Plattform-spezifische Varianten (Story, Feed, LinkedIn, Email)
- Avatar-Evidenz-Tag bei JEDEM Hook

### 3.2 Hook-Kategorien im Detail

**A) Negative Bias (Verlustangst):**
```
Pattern: "Du verlierst [konkret]/[Zeitraum], weil [Pain]"
Pattern: "[Zahl]% der [Zielgruppe] verschwenden [Ressource] an [Problem]"
Pattern: "Jeden Tag den du wartest, kostet dich [konkreter Betrag]"
```
→ Avatar: Anxiety (E), Belief System (G) — hindering beliefs

**B) Curiosity Gap (Informationsdefizit):**
```
Pattern: "Der eine Fehler, den [Zielgruppe] bei [Thema] machen"
Pattern: "Was [Experte/Authority] dir über [Thema] nicht sagt"
Pattern: "Warum [allgemeine Annahme] falsch ist"
```
→ Avatar: Pain (E), JTBD (C), hindering Beliefs (G)

**C) Visual Disruption (Musterunterbrechung):**
```
Pattern: [Visueller Bruch] + "Hör auf, [Verhalten] zu tun."
Pattern: [Kontrast-Split-Screen] "So sieht [Problem] aus. So sieht [Lösung] aus."
Pattern: [Nahaufnahme/Stimme direkt in Kamera] Provokante Behauptung
```
→ Avatar: Belief System (G), Desire & Identity (F)

**D) Specific Result (Greifbarkeit/Beweis):**
```
Pattern: "Wie [Name] in [Zeitraum] [konkretes Ergebnis] erreicht hat"
Pattern: "[Zahl] — das ist der ROI den unsere Kunden im Schnitt sehen"
Pattern: "Vorher: [Pain-State]. Nachher: [Gain-State]. In nur [Zeitraum]."
```
→ Avatar: Desired Outcome (E), Gain (E), Social Proof (H — Testimonials)

### 3.3 Plattform-spezifische Anpassung

| Plattform | Hook-Länge | Visuell | Tonfall |
|-----------|-----------|---------|---------|
| **Story/Reels** | Max. 2 Sekunden | Vollbild, Captions zwingend | Direkt, provokant |
| **Feed (IG/FB)** | 1–3 Sekunden | 4:5 oder 1:1, Thumbnail wichtig | Neugierig, scroll-stoppend |
| **LinkedIn** | 3–5 Sekunden | 16:9, professionell | Datenbasiert, nachdenklich |
| **YouTube Pre-Roll** | 5 Sekunden (Skip-Deadline!) | 16:9, Motion Graphics | Shock oder Curiosity |
| **Email-VSL-Embed** | Sofort zum Wert | Inline, kein Vorspann | Persona-getrieben |
| **Landing Page** | 2–3 Sekunden (Autoplay) | 16:9, Captions | Solution-orientiert |

### 4.2 Captions & Silent Optimization (PFLICHT)

**Captions als Gestaltungselement:**
- Nich nur Untertitel — integraler Bestandteil des Designs
- Animated Captions (Wort-für-Wort-Hervorhebung) für 30%+ bessere Retention
- Brand-Colors in Captions verwenden (aus Brand Strategy)
- Schriftgröße: Minimum 40px bei 1080p (Mobile-First!)
- Kontrast: Weiße Schrift + dunkler Shadow/Outline auf ALLEN Hintergründen

**VTT-Datei Specs:**
```yaml
vtt_specs:
  format: "WebVTT (.vtt)"
  timing_precision: "HH:MM:SS.mmm"
  speaker_id: "Ja — Sprecher-Kennung in Anmerkungen"
  sound_effects: "Ja — [Musik], [SFX: Whoosh], [Pause] als Label"
  language: "de" # BCP 47
  accessibility: "WCAG 2.1 AA konform (Captions ≥95% synchron)"
```

**Visuelle Hinweise für lautlosen Konsum:**
- Key-Points als Text-Overlays (nicht nur gesprochen)
- Emotionale Momente durch visuelle Cues (Icons, Farbwechsel, Zoom)
- Daten/Zahlen als Infographic-Inserts
- CTAs als animierte Buttons ON-SCREEN (nur mündlich reicht nicht)

### Quality (Bewertung 1–5)

| Kriterium | 1 (Schwach) | 3 (Okay) | 5 (Stark) |
|-----------|------------|---------|-----------|
| **Avatar-Evidenz** | Kein Bezug | Teilweise Tags | Jedes Element rückführbar |
| **Hook-Stärke** | Generisch | Brauchbar | Stoppt den Scroll |
| **Emotional Arc** | Flach | Vorhanden | Fühlt sich wie Story an |
| **Segment-Fit** | One-Size-Fits-All | Grob segmentiert | Pro Segment optimiert |
| **Mobile-Readiness** | Desktop-only | Captions da | Full Silent Optimization |
| **Legal Compliance** | Ignoriert | Teilweise geprüft | Alle Checklisten durch |
| **Production-Ready** | Vage Ideen | Spezifiziert | Direkt umsetzbar |

### Was NICHT mehr funktioniert 2026 (Red Flags)

- ❌ Hochglanz-Studioproduktion (wirkt distanziert)
- ❌ Statische Einleitungen / Firmenlogos am Anfang der VSL
- ❌ Isolierte Taktiken ohne System-Kontext
- ❌ Ignorieren mobiler Optimierung (Silent Video!)
- ❌ KI ohne menschliche kreative Oberhoheit (Uncanny Valley)
- ❌ Fake-Urgency ("Nur noch 3 Stunden!" bei Endlos-Loop)
- ❌ Feature-Dumping statt Benefit-Storytelling
- ❌ Eingleisige CTA ohne Objection Handling
- ❌ Voice-Only ohne Captions
- ❌ Auto-generierte Captions ohne Review
